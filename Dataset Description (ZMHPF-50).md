## 2. 数据类别划分

zizhi50 数据集从行为语义上划分为两大类：

### 2.1 ADL（Activities of Daily Living）

表示人体**正常日常行为**，包含四种典型动作：

| 行为类别 | 含义 |
| -------- | ---- |
| sit      | 坐姿 |
| squat    | 蹲姿 |
| stand    | 站立 |
| walk     | 行走 |

> 每类行为均独立建文件夹，并提供多视角同步数据。

------

### 2.2 FALL

表示人体**跌倒行为（异常行为）**：

- 不进行进一步动作细分
- 用于与 ADL 类别形成对比学习

------

## 3. 多视角采集机制

为了增强模型对不同视角的鲁棒性，数据集采用多摄像头同步采集方式：

| 视角编号 | 说明     |
| -------- | -------- |
| camera1  | 主视角   |
| camera2  | 侧视角   |
| camera3  | 斜视角   |
| camera4  | 补充视角 |

> 所有视角数据在时间上保持同步，对应同一动作过程。

------

## 4. 单视角数据组成

每个 `cameraN` 文件夹中包含三类核心数据：

### 4.1 视频数据（Video）

- 存储路径：`Video/`
- 数据内容：原始视频片段
- 数据特点：
  - 每段视频固定为 **50 帧**
  - 可直接用于时序模型输入（如 LSTM / Transformer）

------

### 4.2 帧图像数据（Frame Image）

- 存储路径：`Frame Image/`
- 数据内容：视频逐帧拆解图像
- 作用：
  - 支持图像级处理
  - 可用于关键点检测或CNN特征提取

------

### 4.3 骨骼关键点数据（Synchronized Data）

- 文件格式：`.xlsx`
- 数据来源：YOLO-Pose 模型检测结果
- 数据特点：
  - 与帧图像一一对应
  - 每一帧包含完整人体关键点信息
  - 每个关键点包含：
    - x（横坐标）
    - y（纵坐标）
    - confidence（置信度）

------

## 5. 数据时序特性

- 所有视频均为 **固定长度（50帧）**
- 数据具备天然时间序列结构：

```
视频 → 帧序列 → 骨骼关键点序列
```

> 非常适合用于：
>
> - 时序建模
> - 动作识别
> - 跌倒检测
